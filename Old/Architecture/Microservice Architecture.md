# Microservice Architecture

Мікросервісна архітектура — це підхід до проєктування, при якому додаток складається з набору невеликих, незалежних сервісів, кожен з яких відповідає за окрему частину бізнес-логіки.

## Характеристики
- **Loose Coupling** — сервіси слабо пов'язані між собою і можуть розгортатися незалежно
- **Незалежне масштабування** — кожен сервіс можна масштабувати окремо
- **Технологічна різноманітність** — різні сервіси можуть використовувати різні мови та БД
- **Складність** — потребує інфраструктури для оркестрації, моніторингу та комунікації

## Комунікація між сервісами
- **Синхронна:** REST API, [[gRPC]] (з використанням [[Protobuff]])
- **Асинхронна:** [[RabbitMQ]], Kafka та інші брокери повідомлень

## Інфраструктура
- Контейнеризація через [[Docker]]
- Оркестрація через Kubernetes
- API Gateway для маршрутизації запитів

## Use cases
Великі, складні системи, які потребують масштабованості, незалежного розгортання та роботи великих команд (e-commerce платформи, стрімінгові сервіси, банківські системи).
