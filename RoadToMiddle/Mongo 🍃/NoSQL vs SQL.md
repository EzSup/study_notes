# NoSQL vs SQL

> Порівняння двох підходів до зберігання даних: реляційного (SQL) та нереляційного (NoSQL).

## Ключові відмінності

| Критерій | SQL | NoSQL |
|---|---|---|
| **Модель даних** | Таблиці з рядками та колонками | Документи, ключ-значення, графи, колонки |
| **Схема** | Фіксована (schema-on-write) | Гнучка (schema-on-read) |
| **Мова запитів** | SQL (стандартизована) | Різна для кожної БД (MQL, CQL тощо) |
| **Масштабування** | Вертикальне (потужніший сервер) | Горизонтальне (більше серверів, sharding) |
| **Транзакції** | Повна підтримка ACID | Залежить від БД (MongoDB підтримує ACID) |
| **JOIN-и** | Нативна підтримка | Зазвичай уникаються (embedding замість JOIN) |
| **Узгодженість** | Strong consistency | Eventual consistency (зазвичай) |

## Коли обирати SQL

- Структура даних стабільна і добре визначена
- Потрібні складні JOIN-и між сутностями
- Критична строга узгодженість (фінанси, банкінг)
- Нормалізація важлива для уникнення дублювання

## Коли обирати NoSQL

- Схема даних часто змінюється
- Потрібне горизонтальне масштабування на великі обсяги
- Дані природно вкладені (документи, профілі, каталоги)
- Висока швидкість запису/читання важливіша за складні запити
- Робота з напівструктурованими або поліморфними даними

## CAP-теорема

Будь-яка розподілена система може гарантувати лише 2 з 3 властивостей:

- **C** (Consistency) — всі вузли бачать однакові дані
- **A** (Availability) — кожен запит отримує відповідь
- **P** (Partition tolerance) — система працює при розриві мережі

| БД | Тип | CAP |
|---|---|---|
| PostgreSQL | SQL | CA |
| MongoDB | NoSQL (документна) | CP |
| Cassandra | NoSQL (колонкова) | AP |
| Redis | NoSQL (ключ-значення) | CP |

## Приклад: одна й та ж сутність

### SQL (нормалізовано)
```sql
-- Таблиця users
SELECT u.name, a.city
FROM users u
JOIN addresses a ON u.id = a.user_id
WHERE u.id = 1;
```

### NoSQL (embedded document)
```js
db.users.findOne(
  { _id: 1 },
  { name: 1, "address.city": 1 }
)
```

В NoSQL дані зберігаються разом з об'єктом - не потрібен JOIN, але можливе дублювання.
