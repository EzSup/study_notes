> Дані зберігаються як BSON-документи (Binary JSON) — розширена версія JSON з додатковими типами даних.

## Структура документа

Набір пар `field: value`. Значення може бути будь-яким BSON-типом — вкладений документ, масив, дата, число тощо.

```js
{
  _id: ObjectId("5099803df3f4948bd2f98391"),
  name: { first: "Alan", last: "Turing" },  // вкладений документ
  birth: new Date('Jun 23, 1912'),           // Date
  contribs: ["Turing machine", "Turing test"], // масив рядків
  views: Long(1250000)                       // NumberLong
}
```

## `_id` поле

- Обов'язковий первинний ключ у кожному документі
- Якщо не вказати — драйвер або mongod згенерує `ObjectId` автоматично
- Завжди перше поле в документі
- Не може бути масивом, regex або undefined
- MongoDB автоматично створює унікальний індекс на `_id`

**Варіанти значень:**

| Варіант | Коли використовувати |
|---|---|
| `ObjectId` (дефолт) | Загальний випадок |
| Натуральний унікальний ключ | Якщо вже є (економить індекс) |
| UUID як BinData | Розподілені системи |
| Auto-increment число | Якщо потрібна послідовність |

## Dot notation

Єдиний спосіб звертатись до вкладених полів і елементів масивів у запитах.

```js
// вкладений документ
"name.last"              // → "Turing"
"contact.phone.number"   // → необмежена глибина

// елемент масиву за індексом (0-based)
"contribs.0"             // → "Turing machine"
"contribs.2"             // → "Turingery"
```

## Обмеження

| Обмеження | Значення |
|---|---|
| Максимальний розмір документа | **16 MB** (BSON) |
| Для більших даних | GridFS API |
| Дублікати полів | Не підтримуються — поведінка непередбачувана |
| Назви полів | Не можуть містити `null`-символ |

## Порядок полів

- BSON зберігає поля **впорядковано** (на відміну від JS-об'єктів)
- `{a:1, b:1}` ≠ `{b:1, a:1}` при порівнянні документів
- ⚠️ `$project`, `$addFields`, `$set`, `$unset` можуть змінювати порядок полів — не покладайся на нього в результатах

## Використання структури документа в інших контекстах

```js
// Query filter
{ age: { $gt: 18 }, city: "Kyiv" }

// Update specification
{ $set: { city: "Lviv" }, $inc: { age: 1 } }

// Index specification
{ name: 1, age: -1 }  // 1 = asc, -1 = desc
```
